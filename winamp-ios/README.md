# RetroAmp for iOS

RetroAmp — a classic-style player for iPhone/iPad, built with
[Capacitor](https://capacitorjs.com) and
[Webamp](https://github.com/captbaritone/webamp) — the same MIT-licensed
Winamp 2.9 reimplementation used by the sibling `winamp-mac/` Electron app,
here wrapped in a native iOS shell (WKWebView).

## Features

- Classic Winamp UI in a phone-friendly vertical stack: main player,
  10-band equalizer, playlist, and the Milkdrop (Butterchurn) visualizer
- Touch toolbar: **Add Songs** (native Files picker, multi-select),
  **Load Skin** (any classic `.wsz`), **Milkdrop** (show/hide the
  visualizer)
- Background audio: keeps playing when the app is backgrounded or the
  ringer is on silent (`UIBackgroundModes: audio` +
  `AVAudioSession(.playback)` in `AppDelegate.swift`)
- Lock screen / Control Center "now playing" integration via the Media
  Session API (Webamp's `enableMediaSession`)

## Requirements

- macOS with Xcode (to build/run)
- Node.js 18+

## Setup & run

```sh
cd winamp-ios
npm install          # also copies the webamp bundle into www/lib
npx cap sync ios
npx cap open ios     # opens Xcode; pick a simulator or device and Run
```

The native Xcode project in `ios/` is committed (generated by
`npx cap add ios`, with two hand edits: background-audio entries in
`Info.plist` and the `AVAudioSession` setup in `AppDelegate.swift`).
Capacitor 8 uses Swift Package Manager, so no CocoaPods install is needed.

## Project layout

- `www/` — the web app: `index.html` (toolbar + Webamp container),
  `app.js` (mounts Webamp, wires the toolbar and file inputs), `style.css`
  (safe-area-aware layout, scroll/zoom locking)
- `www/lib/` — the Webamp butterchurn bundle, copied from `node_modules`
  by the `copy-webamp` npm script (gitignored; recreated on `npm install`)
- `ios/` — the native Xcode project
- `capacitor.config.json` — app id `com.docconnect.winampios`, webDir `www`

## Verification status

Built in a Linux sandbox (no macOS/Xcode), so the native build itself was
not run. The web layer was verified headlessly in Chromium at an
iPhone-sized viewport (390×844, touch enabled):

- All four windows render and stack correctly under the toolbar
- Adding a WAV through the toolbar file input appends it to the playlist
  with the correct duration, and playback reaches `PLAYING`
- The Milkdrop toggle opens the visualizer and its WebGL canvas animates

Not verified: the actual iOS build, the native Files picker behavior in
WKWebView, background audio, and lock screen controls. Run it from Xcode
on a simulator or device to confirm.

## Known limitations

- Webamp's UI is mouse-first; taps and drags work, but fine-grained
  gestures (e.g. dragging EQ sliders) can feel fiddly on small screens.
- Skins and playlists are not persisted across launches yet (the Mac app
  persists both; wiring the same through Capacitor Preferences/Filesystem
  is a natural next step).
- No Media Library / CD / video / native plugin support — same constraints
  as the Mac app (see `winamp-mac/README.md`).
